# Лабораторная работа по тестированию ПО

Вариант:

1. Функция `sin(x)`.
2. Программный модуль для работы с B+ деревьями, максимальное количество элементов в ключе `7`.
3. Предметная область по тексту про капитана, охранника, двоих задержанных и мостик.

## 1. Модульное тестирование разложения `sin(x)` в степенной ряд

Реализация находится в `src/main/kotlin/org/example/math/TaylorSin.kt`.

Функция вычисляется по формуле:

`sin(x) = x - x^3 / 3! + x^5 / 5! - ...`

Перед суммированием аргумент приводится к диапазону `[-pi; pi]`, чтобы ряд быстрее сходился.

Выбранное тестовое покрытие:

- `x = 0`: базовый случай.
- `x` около нуля: проверка точности на малых значениях.
- положительные и отрицательные значения: проверка нечётности функции.
- характерные точки `-pi`, `-pi/2`, `-pi/6`, `pi/6`, `pi/2`, `pi`: проверка на опорных значениях.
- большое значение аргумента: проверка нормализации по периоду.
- невалидный `epsilon`: проверка защитного условия.

## 2. Модульное тестирование алгоритма B+ дерева

Реализация находится в `src/main/kotlin/org/example/bplustree/BPlusTree.kt`.

Для тестирования выбраны характерные точки алгоритма:

- `INSERT_START`
- `LEAF_VISIT`
- `LEAF_INSERT`
- `LEAF_OK`
- `LEAF_SPLIT`
- `INTERNAL_VISIT`
- `DESCEND_i`
- `INTERNAL_PROMOTE`
- `INTERNAL_OK`
- `ROOT_SPLIT`
- `SEARCH_START`
- `SEARCH_INTERNAL`
- `SEARCH_DESCEND_i`
- `SEARCH_IN_LEAF`
- `SEARCH_HIT`
- `SEARCH_MISS`

Эталонные сценарии:

1. Простая вставка в пустой лист:
   `INSERT_START -> LEAF_VISIT -> LEAF_INSERT -> LEAF_OK -> INSERT_END`
2. Вставка с переполнением корневого листа:
   `INSERT_START -> LEAF_VISIT -> LEAF_INSERT -> LEAF_SPLIT -> ROOT_SPLIT -> INSERT_END`
3. Вставка в дерево с внутренним узлом:
   `INSERT_START -> INTERNAL_VISIT -> DESCEND_1 -> LEAF_VISIT -> LEAF_INSERT -> LEAF_OK -> INSERT_END`
4. Успешный поиск:
   `SEARCH_START -> SEARCH_INTERNAL -> SEARCH_DESCEND_1 -> SEARCH_IN_LEAF -> SEARCH_HIT`
5. Неуспешный поиск:
   `SEARCH_START -> SEARCH_INTERNAL -> SEARCH_DESCEND_1 -> SEARCH_IN_LEAF -> SEARCH_MISS`

В тестах фактическая последовательность сравнивается с эталонной через `assertContentEquals`.

## 3. Доменная модель и тестовое покрытие

Реализация находится в `src/main/kotlin/org/example/domain/BridgeScene.kt`.

Сущности доменной модели:

- `Captain`
- `Guard`
- `Detainee`
- `Notebook`
- `BridgeScene`
- перечисления `DoorState` и `Location`

Правила предметной области:

- в описанной сцене участвуют ровно двое задержанных;
- до действий охранника задержанные находятся на мостике;
- после действий охранника оба задержанных перемещаются в коридор;
- после этого стальная дверь закрыта;
- капитан остаётся один только если дверь закрыта и оба задержанных выведены;
- капитан может заносить фрагменты стихов в записную книжку.

Тестовое покрытие:

- состояние сцены до выведения;
- состояние сцены после выведения;
- запись стихотворного фрагмента в книжку;
- нарушение ограничения на число задержанных.
